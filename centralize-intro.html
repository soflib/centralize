Flujo de operación

CÓMO FUNCIONA

Cada acción queda registrada permanentemente en la blockchain de Solana. No hay base de datos central que pueda ser alterada.

01
REGISTRO
Empresas y proveedores se registran con documentos hasheados. La membresía queda pendiente hasta confirmar pago.
02
VERIFICACIÓN
El administrador activa membresías tras confirmar pago fiat. El hash del comprobante queda on-chain.
03
LICITACIÓN
Empresas con membresía activa publican solicitudes privadas. Solo proveedores verificados pueden postularse.
04
CALIFICACIÓN
Tras completar el servicio, la empresa califica al proveedor. El historial es público e inmutable.

Instrucciones

SETUP LOCAL

Sigue estos pasos en orden para levantar el entorno local completo.

SETUP INICIAL
# Instalar Solana
curl --proto '=https' --tlsv1.2 -sSfL \
  https://solana-install.solana.workers.dev | bash

export PATH="$HOME/.local/share/solana/\
install/active_release/bin:$PATH"

# Crear keypair (si no existe)
solana address || solana-keygen new \
  --outfile ~/.config/solana/id.json \
  --no-bip39-passphrase

solana config set --url http://localhost:8899
⚠️ Solo la primera vez. Si ya tienes keypair, sáltalo.
COMPILAR CONTRATO
cd program/

# Primera compilación
anchor build

# Generar keypair fijo del programa
solana-keygen new \
  --outfile target/deploy/centralize-keypair.json \
  --no-bip39-passphrase --force

# Ver el Program ID permanente
solana-keygen pubkey \
  target/deploy/centralize-keypair.json

# Actualizar en: lib.rs, Anchor.toml, .env
anchor keys sync
anchor build
⚠️ El Program ID generado es permanente. Ponlo en los 3 archivos.
LEVANTAR DOCKER
cd infrastructure/

make dev

# Esperar ~15 segundos...

# Airdrop SOL al validator local
solana airdrop 10 --url http://localhost:8899
solana airdrop 10 --url http://localhost:8899
solana airdrop 10 --url http://localhost:8899
⚠️ Nunca uses "docker compose down" o perderás el ledger.
DESPLEGAR Y USAR
cd program/

# Desplegar contrato
anchor program deploy \
  target/deploy/centralize.so \
  --provider.cluster localnet

# Abrir el HTML de la consola API
# → Click en "Initialize Platform" (naranja)
# → Registrar empresa
# → Registrar proveedor
# → ¡Listo!
⚠️ "Initialize Platform" se llama UNA SOLA VEZ después de cada deploy.
