  source ~/.bashrc

6. DEPLOY
--------------------------------------------------------------------------------

Comando (--use-rpc necesario por incompatibilidad de versiones CLI/validator):
  solana program deploy target/deploy/certchain.so --use-rpc

Resultado:
  Program Id: 326Z4n9fXoHK97GeGEYBy2tAhaqvexd7fueNB9Vv4c3t
  Signature:  5kUa9LHefTENpUnisRzswTcrYgHGPa33UhRkbJfiV9W5zRninbwnrZXGryCgau63Qg2XmYtp3AEnYA5KuQjUfpNB

Por qué --use-rpc:
  El CLI Agave 3.1.12 usa TPU (UDP) para enviar transacciones.
  El validator 1.18.26 (legacy) no expone los líderes de forma compatible.
  --use-rpc fuerza el envío por HTTP/RPC en lugar de UDP/TPU.

Verificar el programa desplegado:
  solana program show 326Z4n9fXoHK97GeGEYBy2tAhaqvexd7fueNB9Vv4c3t

8. INVOCAR EL CONTRATO (Node.js)
--------------------------------------------------------------------------------

Instalar dependencias:
  cd ~/certora
  npm init -y
  npm install @solana/web3.js

Archivo: invoke.mjs

  import {
    Connection, PublicKey, Transaction,
    TransactionInstruction, sendAndConfirmTransaction, Keypair
  } from "@solana/web3.js";
  import { readFileSync } from "fs";
  import { homedir } from "os";

  const connection = new Connection("http://localhost:8899", "confirmed");
  const programId = new PublicKey("326Z4n9fXoHK97GeGEYBy2tAhaqvexd7fueNB9Vv4c3t");

  const keypairData = JSON.parse(
    readFileSync(`${homedir()}/.config/solana/id.json`, "utf8")
  );
  const payer = Keypair.fromSecretKey(Uint8Array.from(keypairData));

  const instruction = new TransactionInstruction({
    programId,
    keys: [],
    data: Buffer.alloc(0),
  });

  const tx = new Transaction().add(instruction);
  const sig = await sendAndConfirmTransaction(connection, tx, [payer]);
  console.log("Transacción:", sig);

Ejecutar:
  node invoke.mjs
